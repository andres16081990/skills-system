## 4. Análisis de causa raíz

### Hipótesis

- [x] **H1 — Campo `b64` con URL en lugar de base64:** En `buildLink` (`createPolicyAdapter.ts`), para el tipo S3 se envía `b64: p.linkUrl`. `p.linkUrl` almacena la URL del archivo ya subido a S3 (ej. `https://assets.app...`), por lo que el campo `b64` llegaba al backend con una URL en lugar de un string base64, causando rechazo en la validación del servidor. ✅ CONFIRMADA
- [ ] **H2 — HTTP method incorrecto**
- [ ] **H3 — `typePortal` con valor incorrecto**
- [ ] **H4 — `channels: []` rechazado por el backend**

### Causa confirmada

`buildLink` en `createPolicyAdapter.ts` asignaba `b64: p.linkUrl` para el tipo S3, pero `p.linkUrl` contiene la URL del archivo ya subido a S3. El backend espera la URL en el campo `url`, no en `b64`.

```
// ANTES (incorrecto):
return { name: p.linkName, url: "", b64: p.linkUrl, type: "S3" };
//  url vacío, b64 con URL S3 → backend rechaza el request

// DESPUÉS (correcto):
return { name: p.linkName, url: p.linkUrl, type: "S3" };
//  url con URL S3, b64 omitido
```

**Archivo raíz:** `src/presentation/adminSettings/TermsAndConditions/helpers/createPolicyAdapter.ts`
**Línea(s):** 28–33

## 5. Plan de fix

### Estrategia

Corregir `buildLink` en `createPolicyAdapter.ts` para que la URL S3 se asigne al campo `url` y se omita `b64`.

### Cambios requeridos

| # | Archivo | Cambio | Principio |
|---|---------|--------|-----------|
| 1 | `src/.../helpers/createPolicyAdapter.ts` | En `buildLink`, caso S3: mover `p.linkUrl` de `b64` a `url` y eliminar el campo `b64` | SRP |

### Restricciones

- ❌ No modificar `useCreatePolicie` — el bug está en el adapter, no en el hook
- ❌ No alterar `PolicyLinkRequest` — `b64` ya es opcional (`b64?: string`), solo se deja de usar
- ✅ `formValuesToPolicyRequest` tiene un único consumidor, modificación directa permitida

### Cambios realizados

```diff
// src/presentation/adminSettings/TermsAndConditions/helpers/createPolicyAdapter.ts
// función buildLink — caso S3

- return {
-   name: p.linkName,
-   url: "",
-   b64: p.linkUrl,

- };
+ return {
+   name: p.linkName,
+   url: p.linkUrl,
+   type: "S3",
+ };
```

### Lecciones aprendidas

- **Patrón del bug:** El campo `b64` de `PolicyLinkRequest` existe para enviar un PDF en base64 al backend, pero el flujo real almacena la URL S3 del archivo ya subido en `p.linkUrl`. Nunca se debió usar `b64` para una URL.
- **Skill relacionada:** `components/useCreatePolicie.md`
- **Regla nueva:** Para links tipo S3, `p.linkUrl` siempre contiene la URL del archivo ya subido. Debe ir en el campo `url`, nunca en `b64`.
