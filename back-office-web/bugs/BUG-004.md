# 🐛 Bugfix Spec: Regex de validación Collector excluye punto y coma; mensaje de error poco descriptivo

> **ID:** BUG-004
> **Fecha:** 2026-02-25
> **Proyecto:** back-office-web
> **Prioridad:** medium
> **Estado:** resolved
> **Relacionado con:** BUG-002, BUG-003

---

## 1. Resumen

`collectorTitleRegex` y `collectorContentRegex` en `formValidator.ts` no permiten punto (`.`) ni coma (`,`), bloqueando textos válidos. Además los mensajes de error en i18n enumeran caracteres específicos en lugar de dar un mensaje general.

---

## 2. Contexto

### Archivos involucrados

| Archivo | Rol |
|---------|-----|
| `src/presentation/adminSettings/TermsAndConditions/helpers/formValidator.ts` | Define los regex y los usa en `validateTitle` y `validateContent` |
| `src/i18n/languages/es.json` | Contiene los mensajes `titleInvalidChars` y `contentInvalidChars` |

### Regex actuales (a cambiar)

```ts
// formValidator.ts
const collectorTitleRegex   = /^[a-zA-ZÀ-ÿ\s&\-_]+$/;
const collectorContentRegex = /^[a-zA-ZÀ-ÿ\s&\-]+$/;
```

### Mensajes actuales (a cambiar)

```json
"titleInvalidChars":   "El título solo permite letras, tildes, &, - y _",
"contentInvalidChars": "El contenido solo permite letras, tildes, & y -"
```

---

## 3. Reproducción

### Pasos

1. Ir a `/configuracion/terminos-y-condiciones/collector`
2. Crear política con título `"Política de datos, versión 1.0"`
3. Intentar guardar

### Comportamiento actual

Error de validación: `"El título solo permite letras, tildes, &, - y _"` — bloqueando un texto perfectamente válido que contiene `.` y `,`.

### Comportamiento esperado

- Título con `.` y `,` pasa la validación sin error
- Si hay un carácter realmente inválido (ej. `!`, `@`, `#`), el mensaje muestra: `"No se permiten caracteres especiales"`

---

## 4. Análisis de causa raíz

### Causa confirmada

Los regex no incluyen `\.` ni `,` en su set de caracteres permitidos.

```
/^[a-zA-ZÀ-ÿ\s&\-_]+$/   ← falta ., ,
/^[a-zA-ZÀ-ÿ\s&\-]+$/    ← falta ., ,
```

**Archivo raíz:** `src/presentation/adminSettings/TermsAndConditions/helpers/formValidator.ts`
**Líneas:** 6–7

---

## 5. Plan de fix

### Cambios requeridos

| # | Archivo | Cambio |
|---|---------|--------|
| 1 | `formValidator.ts` | Agregar `.,` al set de caracteres de ambos regex |
| 2 | `es.json` | Reemplazar mensajes específicos por texto general |

### Regex nuevos

```ts
const collectorTitleRegex   = /^[a-zA-ZÀ-ÿ\s&\-_.,]+$/;
const collectorContentRegex = /^[a-zA-ZÀ-ÿ\s&\-.,]+$/;
```

### Mensajes nuevos

```json

"contentInvalidChars": "No se permiten caracteres especiales en el contenido"
```
