# 🐛 Bugfix Spec: Regex de validación Collector excluye caracteres numéricos

> **ID:** BUG-005
> **Fecha:** 2026-02-25
> **Proyecto:** back-office-web
> **Prioridad:** medium
> **Estado:** resolved
> **Relacionado con:** BUG-004

---

## 1. Resumen

`collectorTitleRegex` y `collectorContentRegex` en `formValidator.ts` no permiten dígitos (`0-9`), bloqueando textos válidos como `"Política versión 2"` o contenido con referencias numéricas como `"artículo 3"`.

---

## 2. Contexto

### Archivos involucrados

| Archivo | Rol |
|---------|-----|
| `src/presentation/adminSettings/TermsAndConditions/helpers/formValidator.ts` | Define los regex y los usa en `validateTitle` y `validateContent` |
| `src/presentation/adminSettings/TermsAndConditions/helpers/__test__/formValidator.test.ts` | Tests que actualmente asumen que los números son inválidos |

### Regex actuales (a cambiar)

```ts
// formValidator.ts — estado post BUG-004
const collectorTitleRegex   = /^[a-zA-ZÀ-ÿ\s&\-_.,]+$/;
const collectorContentRegex = /^[a-zA-ZÀ-ÿ\s&\-.,]+$/;
```

---

## 3. Reproducción

### Pasos

1. Ir a `/configuracion/terminos-y-condiciones/collector`
2. Crear política con título `"Política versión 2"`
3. Intentar guardar

### Comportamiento actual

Error de validación: `"No se permiten caracteres especiales en el título"` — bloqueando un texto perfectamente válido que contiene dígitos.

### Comportamiento esperado

- Título con dígitos como `"Política versión 2"` pasa la validación sin error
- Contenido con dígitos como `"Ver artículo 3, inciso 1.2"` pasa la validación sin error
- Si hay un carácter realmente inválido (ej. `!`, `@`, `#`), el mensaje sigue mostrando: `"No se permiten caracteres especiales"`

---

## 4. Análisis de causa raíz

### Causa confirmada

Los regex no incluyen `0-9` en su set de caracteres permitidos.

```
/^[a-zA-ZÀ-ÿ\s&\-_.,]+$/   ← falta 0-9
/^[a-zA-ZÀ-ÿ\s&\-.,]+$/    ← falta 0-9
```

**Archivo raíz:** `src/presentation/adminSettings/TermsAndConditions/helpers/formValidator.ts`
**Líneas:** 6–7

---
