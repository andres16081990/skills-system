# 🐛 Bugfix Spec: Input URL muestra base64 del PDF al cambiar de tipo S3 a URL

> **ID:** BUG-006
> **Fecha:** 2026-02-25
> **Proyecto:** back-office-web
> **Prioridad:** medium
> **Estado:** resolved

---

## 1. Resumen

Al cambiar el radio button de `TypePolicie.S3` (PDF cargado) a `TypePolicie.URL`, el `OutlinedInput` del campo URL aparece pre-llenado con el base64 del PDF. El campo URL debería estar vacío al cambiar de tipo.

---

## 2. Contexto

### Archivos involucrados

| Archivo | Rol |
|---------|-----|
| `src/presentation/adminSettings/TermsAndConditions/components/AttachmenComponent/hooks/usePolicyAttachmentCollapse.ts` | Maneja `handleTypeChange` y el estado `link` |
| `src/presentation/adminSettings/TermsAndConditions/components/AttachmenComponent/context/context.tsx` | Provee `link` / `setLink` y sincroniza con `initialLink` |

### Flujo que produce el bug

```
1. Usuario selecciona TypePolicie.S3 y carga un PDF
2. Usuario hace clic en "Aplicar"
   → handleApplyClick() convierte el PDF a base64
   → onLinkChange(b64) → actualiza linkUrl en Formik (parent)
   → context: initialLink pasa a ser el base64
3. useEffect en context.tsx: setLink(initialLink) → link = base64
4. Usuario cambia el radio a TypePolicie.URL
   → handleTypeChange(TypePolicie.URL) → solo llama setType()
   → link sigue siendo el base64
5. OutlinedInput value={link} muestra el base64 del PDF ← BUG
```

### Causa raíz confirmada

`handleTypeChange` en `usePolicyAttachmentCollapse.ts` solo cambia el tipo pero **no resetea el estado `link`** del contexto. Como el contexto sincroniza `link` con `initialLink` (que ya fue actualizado con el base64 al aplicar el PDF), el campo URL queda contaminado.

```ts
// usePolicyAttachmentCollapse.ts — línea 42
const handleTypeChange = (newType: AttachmentType) => {
  setType(newType);       // ← solo cambia el tipo
  onTypeChange(newType);  // ← no resetea link
};
```

---

## 3. Reproducción

### Pasos

1. Ir a `/configuracion/terminos-y-condiciones/collector`
2. Abrir el `ButtonCard` de adjunto en una política
3. Seleccionar `TypePolicie.S3`, cargar un PDF y hacer clic en "Aplicar"
4. Volver a abrir el `ButtonCard` y cambiar el radio a `TypePolicie.URL`

### Comportamiento actual

El `OutlinedInput` del campo URL aparece con el string base64 del PDF.

### Comportamiento esperado

Al cambiar a `TypePolicie.URL`, el campo URL aparece vacío.

---

## 4. Plan de fix

### Cambio requerido

Solo un archivo: `usePolicyAttachmentCollapse.ts`, función `handleTypeChange`.

Al cambiar de tipo, resetear `link` en el contexto llamando `setLink("")`. **No llamar `onLinkChange("")`** — eso propagaría el vacío al formulario Formik y rompería el guardado.

```ts
// ANTES
const handleTypeChange = (newType: AttachmentType) => {
  setType(newType);
  onTypeChange(newType);
};

// DESPUÉS
const handleTypeChange = (newType: AttachmentType) => {
  setType(newType);
  setLink("");
  onTypeChange(newType);
};
```

### Restricciones críticas

- ❌ No llamar `onLinkChange("")` en `handleTypeChange` → rompe el guardado
- ❌ No modificar `handleApplyClick` → el guardado funciona correctamente
- ❌ No modificar `context.tsx` ni el `useEffect` que sincroniza `initialLink`
- ✅ Solo agregar `setLink("")` dentro de `handleTypeChange`

## 5. Criterios de aceptación

- [ ] Cargar PDF en S3 → aplicar → cambiar a URL → escribir URL → aplicar → `linkUrl` del form tiene la URL, no el base64
- [ ] Cargar PDF en S3 → aplicar → no cambia tipo → `linkUrl` del form sigue siendo el base64 (sin regresión)
- [ ] Cambiar a URL → escribir URL → cambiar a S3 → volver a URL → campo URL vacío (limpio en cada cambio)
