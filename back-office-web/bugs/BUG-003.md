# 🐛 Bugfix Spec: Claves i18n faltantes para validaciones de caracteres en políticas

> **ID:** BUG-003
> **Fecha:** 2026-02-25
> **Proyecto:** back-office-web
> **Prioridad:** medium
> **Estado:** resolved
> **Relacionado con:** BUG-002

---

## 1. Resumen

Las claves `validations.titleInvalidChars` y `validations.contentInvalidChars` usadas en `formValidator.ts` (fix de BUG-002) no existen en el JSON de i18n, por lo que los mensajes de error se muestran como la clave cruda en lugar del texto en español.

---

## 2. Contexto

### Archivos involucrados

| Archivo | Rol |
|---------|-----|
| `src/i18n/languages/es.json` | Único archivo de traducciones del proyecto |
| `src/presentation/adminSettings/TermsAndConditions/helpers/formValidator.ts` | Consume las claves vía `translate("validations.titleInvalidChars")` y `translate("validations.contentInvalidChars")` |

### Ubicación exacta en el JSON

```
properties.termsAndConditions.pages.main.validations
```

Claves vecinas actuales (líneas ~785–799):
```json
"validations": {
  "duplicateTitle": "...",
  "atLeastOnePolicy": "...",
  "titleRequired": "...",
  "contentRequired": "...",
  "contentMax": "...",
  "channelRequired": "...",
  "linkNameRequired": "...",
  "urlRequired": "...",
  "urlInvalid": "...",
  "filesNotAllowedForUrl": "...",
  "fileRequired": "...",
  "fileLinkNameRequired": "...",
  "attachmentTypeInvalid": "...",
  "titleDuplicate": "..."
  ← insertar aquí las dos claves faltantes
}
```

---

## 3. Reproducción

### Pasos

1. Ir a `/configuracion/terminos-y-condiciones/collector`
2. Crear una política con caracteres inválidos en el título (ej. `Política 123`)
3. Intentar guardar

### Comportamiento actual

El mensaje de error muestra la clave cruda:
```
validations.titleInvalidChars
validations.contentInvalidChars
```

### Comportamiento esperado

```
El título solo permite letras, tildes, &, - y _
El contenido solo permite letras, tildes, & y -
```

---

## 4. Análisis de causa raíz

### Causa confirmada

Las claves fueron referenciadas en `formValidator.ts` (BUG-002) pero nunca se agregaron al JSON de traducciones.

**Archivo raíz:** `src/i18n/languages/es.json`
**Sección:** `properties.termsAndConditions.pages.main.validations`

---

## 5. Plan de fix

### Cambios requeridos

| # | Archivo | Cambio |
|---|---------|--------|
| 1 | `src/i18n/languages/es.json` | Agregar `"titleInvalidChars"` y `"contentInvalidChars"` dentro de `validations`, después de `"attachmentTypeInvalid"` |

### Textos a agregar

```json
"titleInvalidChars": "El título solo permite letras, tildes, &, - y _",
"contentInvalidChars": "El contenido solo permite letras, tildes, & y -"
```

### Restricciones

- ❌ No modificar claves existentes
- ❌ No cambiar la estructura del JSON
- ✅ Insertar las dos claves nuevas dentro del objeto `validations` existente

---

## 6. Criterios de aceptación

- [ ] Al ingresar caracteres inválidos en el título (modo Collector) se muestra: `"El título solo permite letras, tildes, &, - y _"`
- [ ] Al ingresar caracteres inválidos en el contenido (modo Collector) se muestra: `"El contenido solo permite letras, tildes, & y -"`
- [ ] El JSON sigue siendo válido (sin errores de parseo)

## 8. Resolución

### Fecha de resolución

2026-02-25

### Cambios realizados

```diff
// src/i18n/languages/es.json — dentro de validations

  "attachmentTypeInvalid": "Tipo de adjunto inválido",
+ "titleInvalidChars": "El título solo permite letras, tildes, &, - y _",
+ "contentInvalidChars": "El contenido solo permite letras, tildes, & y -",
  "titleDuplicate": "Ya existe una política con este nombre"
```
