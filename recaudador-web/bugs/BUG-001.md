# BUG-001

## Descripción
Al navegar desde el dashboard hacia PaymentDocuments, se están
disparando 3 peticiones a getList de documentos en lugar de una sola.

## Pasos para reproducir
1. Estar en el dashboard
2. Navegar a Payment Documents
3. Observar en Network tab del navegador — aparecen 3 llamadas a getList

## Comportamiento esperado
Al entrar a PaymentDocuments debe realizarse un único dispatch
con los filtros del state de Redux. No debe haber llamadas duplicadas.

## Componentes a revisar
- PaymentDocuments

## Lo que el agente debe investigar
- Analiza el action o thunk que llama a getList e identifica su nombre exacto
- Busca todos los lugares del componente y sus hijos donde se hace
  dispatch de ese action (useEffect, eventos, hooks)
- Revisa si hay componentes hijos dentro de PaymentDocuments que
  también estén haciendo dispatch del mismo action
- Revisa el estado de Redux relacionado: qué filtros existen en el
  state y cómo se están leyendo actualmente
- Identifica si el problema es: múltiples useEffect, múltiples
  componentes despachando, o dependencias incorrectas en un useEffect

## Solución esperada
Un único dispatch al montar el componente, usando los filtros
actuales del state de Redux. Sin llamadas duplicadas.

## Contexto adicional
El bug ocurre específicamente al venir desde el dashboard.
No se sabe aún si es reproducible desde otras rutas.
