# BUG-002

## Descripción
En MassiveSelectionFilter, cuando se abre el modal de eliminación y el filtro
activo es "por vencerse", el adaptador getStateFilterValue construye el request
como si la bandera fromDashboardCardSelected estuviera activa, pero no lo está.

## Pasos para reproducir
1. Aplicar el filtro "por vencerse" en la lista de documentos
2. Seleccionar documentos de forma masiva
3. Abrir el modal de eliminación
4. Observar el request generado por getStateFilterValue — los estados
   vienen incorrectos como si fromDashboardCardSelected fuera true

## Comportamiento esperado
Cuando la acción viene del modal de eliminación con filtro "por vencerse",
getStateFilterValue debe construir el request con los estados correctos
sin activar la lógica de fromDashboardCardSelected.

Cuando viene del dashboard con filtro "por vencerse", debe mantener el
comportamiento actual MÁS agregar el estado adicional correspondiente
a "por vencerse".

## Componentes a revisar
- MassiveSelectionFilter
- PaymentDocumentMain
- DocumentFilters
- getStateFilterValue (adaptador)

## Lo que el agente debe investigar
- Analiza getStateFilterValue y entiende cómo evalúa la bandera
  fromDashboardCardSelected para construir los estados del request
- Identifica en qué condición se activa esa lógica actualmente
- Busca cómo llega el contexto de "eliminación desde modal" al adaptador
- Verifica cuántos lugares usan getStateFilterValue en el proyecto
  antes de tocar cualquier cosa (regla global)
- Si getStateFilterValue es compartido, NO lo modifiques —
  crea una versión independiente para el caso de eliminación

## Solución esperada
El adaptador debe distinguir tres escenarios:

1. Viene del dashboard CON filtro "por vencerse" → comportamiento actual
   + agregar el estado adicional de "por vencerse"

2. Viene del dashboard CON otro filtro → comportamiento actual intacto

3. Viene del modal de eliminación CON filtro "por vencerse" → construir
   el request con los estados correctos SIN activar lógica del dashboard

## Contexto adicional
El problema está aislado al flujo de eliminación masiva con el filtro
"por vencerse" activo. No afecta otros filtros ni el flujo del dashboard.
